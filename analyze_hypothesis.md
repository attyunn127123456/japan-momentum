
## ⚠️ 実装制約（必ず守ること）

### 使えるデータ
- **OHLCV のみ**: 始値(Open)・高値(High)・安値(Low)・終値(Close/AdjC)・出来高(Volume)
- 業種コード（J-Quants equities/masterから取得済み）

### 使えないデータ（J-Quants 403 エラーで取得不可）
- fins/statements（財務諸表）
- fins/dividend（配当）以外のfinsエンドポイント
- short_selling（空売り残高）
- trades_spec（投資主体別売買）

### 新ファクターの条件
1. OHLCVと業種コードのみで計算できること
2. 既存ファクターと相関が低い新情報を追加すること
3. 数式が透明であること（MLなし）

## 最重要ゴール
日経+98%の期間で+300%（年率約60%）を取ること。
シャープ最大化ではなく、リターン最大化（max_dd < 40%制約）が目標。

## 新しい仮説の方向性
- 集中投資（top_n=3〜5）の探索
- 強いトレンドの早期捕捉と長期保有
- 「安定してるが遅い」ファクターより「速くて強い」ファクターを重視

# 改善ループの設計思想

## ループの構造
```
① 仮説生成（opus）: 新ファクターの提案
② ファクター実装: optimize.py の precompute() に追加
③ GA重み最適化: 全ファクターをパラメータとしてGA探索
④ ファクター整理: weight≒0のものを削除、残ったものがシグナルライブラリ
⑤ また①へ
```

## あなた（opus）の役割
**新しいファクター候補を提案すること。**
重みの調整・パラメータのチューニングはGAが自動でやる。
あなたは「何を測るか」という設計を提案する。

## 仮説の書き方（重要）
仮説 = 「新しいファクターを1つ追加する提案」
- 具体的な数値（params）より、**どんな情報を使うか**が重要
- 重みはGAが最適化するので `0.1〜0.2` のプレースホルダーでOK
- 「重みを変える」だけの仮説はNG（それはGAの仕事）
- 「ユニバースをハードフィルターで絞る」仮説はNG（過去に全て失敗）

## 現在のシグナルライブラリ（active）
以下のファクターが `optimize.py` に実装済み・GA探索対象：
- `ret_w`: 80日リターン
- `rs_w`: 日経対比相対強度
- `green_w`: 陽線率
- `smooth_w`: 上昇の滑らかさ
- `high52_w`: 52週高値への近さ ← 最重要（w=0.35）
- `resilience_w`: 下落相場での耐性
- `omega_w`: 上昇速度÷下落幅（非対称性）← 新規追加済み
- `short_momentum_w`: 直近5-10日リターン ← 新規追加済み

## 次に追加すべきファクター候補（優先度順）

### 高優先度
1. **クラスターブースト（`cluster_boost_w`）**
   - 直近60日の価格相関からK-meansで自動テーマ分類
   - 強いクラスターの銘柄にブーストをかける
   - 実装: 月1回クラスタリング → クラスター平均モメンタムをスコアに加算
   
2. **直近スキップリターン（`ret_skip_w`）**
   - 12ヶ月リターンだが直近1ヶ月をスキップ（Jegadeesh-Titman標準）
   - price[t-5] / price[t-85] で計算（直近5日スキップ）
   - 世界中で実証済みの短期反転回避

3. **出来高確認（`volume_confirm_w`）**
   - 直近5日出来高 / 60日平均出来高のランク
   - 高値更新時に出来高も増えている = 本物のブレイクアウト

### 中優先度
4. **残差モメンタム（`residual_w`）**
   - 業種平均リターンを除いた銘柄固有のアルファ
   - 33業種コードをJ-Quantsから取得して計算

5. **業種ブースト（`sector_boost_w`）**
   - 強い業種（33業種分類）の銘柄にスコア加算
   - 業種全体のモメンタムを個別銘柄スコアに反映

## 現在のベースライン
- sharpe=2.53, lb=80, top_n=10, weekly
- ret_w=0.2, rs_w=0.2, green_w=0.125, smooth_w=0.125, high52_w=0.35
- universe: 全銘柄（時価総額フィルターなし）

## 失敗パターン（繰り返さない）
- ユニバースをハードフィルターで絞る → 全て大幅悪化
- lookback/top_nの変更だけ → 効果なし
- 重みの微調整だけ → GAの仕事、仮説ではない

## 実装が必要なもの
仮説に `factor_to_add` が含まれる場合、パンダが `optimize.py` への実装を手配する。
既存ファクター（omega_w, short_momentum_wなど）はGAが自動探索するので実装不要。
